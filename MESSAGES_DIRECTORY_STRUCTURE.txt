i18n Translation Files - Directory Structure
=============================================

messages/
├── en.json                              28 KB  [ORIGINAL - KEEP AS FALLBACK]
├── es.json                              30 KB  [ORIGINAL - KEEP AS FALLBACK]
│
├── common/                              [LOADED ON EVERY PAGE - ~1 KB]
│   ├── en.json                         0.9 KB
│   └── es.json                         1.0 KB
│   └── Contains: loader, nav, theme, footer
│
├── home/                               [HOMEPAGE ONLY - ~10 KB]
│   ├── en.json                        9.8 KB
│   └── es.json                         10 KB
│   └── Contains: hero, services, portfolio, faq, process, team, contactForm, features
│
└── pages/                              [PAGE-SPECIFIC - VARIES BY ROUTE]
    ├── payments/
    │   ├── en.json                    3.0 KB
    │   └── es.json                    3.1 KB
    │   └── Route: /payments
    │
    ├── privacy/
    │   ├── en.json                    3.3 KB
    │   └── es.json                    3.7 KB
    │   └── Route: /privacy
    │
    ├── terms/
    │   ├── en.json                    3.5 KB
    │   └── es.json                    3.9 KB
    │   └── Route: /terms
    │
    └── uiux-design/
        ├── en.json                    4.3 KB
        └── es.json                    4.7 KB
        └── Route: /servicios/diseno-uiux


LOADING LOGIC
=============

route = /                    -> load: common + home
route = /payments            -> load: common + pages/payments
route = /privacy             -> load: common + pages/privacy
route = /terms               -> load: common + pages/terms
route = /servicios/diseno-uiux -> load: common + pages/uiux-design
route = (other)              -> load: common + home (fallback)

TOTAL IMPACT
============

Old Structure:
  - 2 files per language
  - 60 KB loaded per page
  - 100K users/month = 11.98 GB bandwidth

New Structure:
  - 10 files per language (organized by namespace)
  - 10.7 KB average per page (some as low as 3.9 KB)
  - 100K users/month = 1.28 GB bandwidth
  - SAVINGS: 10.7 GB/month (89% reduction)


HOW TO USE
==========

1. Copy optimized config:
   cp i18n/request-optimized.ts i18n/request.ts

2. Test locally:
   npm run dev

3. Check DevTools Network tab:
   - Should see smaller JSON file sizes
   - Homepage ~10.7 KB
   - Payment pages ~3.9 KB

4. Deploy as usual:
   git add . && git commit -m "Optimize i18n" && git push

No component changes needed. Everything backward compatible.

MAINTENANCE GUIDE
=================

Adding a new page?
  1. mkdir -p messages/pages/my-new-page
  2. Create my-new-page/en.json and my-new-page/es.json
  3. Add route mapping to i18n/loader.ts:
     '/my-route': ['common', 'pages/my-new-page']
  4. Done! No component changes needed

Updating translations?
  1. Edit the file in messages/{namespace}/{locale}.json
  2. Refresh browser during dev
  3. Deploy as usual

Checking file sizes?
  ls -lh messages/**/*.json

Validating structure?
  npm run check:i18n  (see I18N_IMPLEMENTATION_GUIDE.md)

SUPPORT
=======

Q: Do I need to change my components?
A: No! All components continue using useTranslations() normally.

Q: Is this backward compatible?
A: Yes! Old messages/en.json and messages/es.json are still present.

Q: What if a route isn't in the mapping?
A: Falls back to ['common', 'home'] namespace automatically.

Q: Can I still use the monolithic files?
A: Yes! Just keep using the old i18n/request.ts if preferred.

Q: How do I add a new page with translations?
A: Create new namespace in messages/pages/my-page/ and update i18n/loader.ts

Q: Will my users notice anything?
A: No breaking changes. Some may experience slightly faster page loads.

For more details, see I18N_IMPLEMENTATION_GUIDE.md
